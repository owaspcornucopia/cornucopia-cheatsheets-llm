# C3 — Access Source Code or Decompile Binaries

## Non-Applicable

This threat does not apply to this application.

## Reasoning

The C3 card describes accessing source code, decompiling, or debugging to understand how the application works and discover secrets.

This application is a server-side Python API. There are no client-side binaries, mobile apps, desktop applications, or distributed executables that an attacker could decompile or reverse-engineer. The source code:

- Runs only on the server
- Is not distributed to clients
- Is not compiled into binaries that could be decompiled

The concern about secrets in source code (hardcoded tokens) is covered by CRJ and AT5, but that relates to repository access — not decompilation of distributed software.
