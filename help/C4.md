# C4 — Actions Cannot Be Attributed

## Non-Applicable

This threat does not apply to this application.

## Reasoning

The C4 card describes performing an action that cannot be attributed to the actor (non-repudiation failure).

While this application has severe logging deficiencies (covered by VEA, ATA, AZA, SMA), the C4 card specifically addresses non-repudiation — the concept that a user should not be able to deny having performed an action.

Non-repudiation is typically relevant for:

- Financial transactions (payments, transfers)
- Legal agreements (digital signatures, contracts)
- Regulatory actions (approvals, sign-offs)

This application performs read-only fraud investigations (queries). It does not execute financial transactions, sign agreements, or perform irrevocable actions that require non-repudiation. The logging gap is already comprehensively covered by the Ace cards in each category.
